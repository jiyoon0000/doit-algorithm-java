# chapter02. 코드의 논리 오류를 어떻게 잡을까?

# 🐞 Debugging

## 1. 정의
디버깅은 프로그램에서 발생하는 **논리 오류를 찾아 수정하는 과정**이다.  
문법 오류는 컴파일러가 잡아주지만, 논리 오류는 직접 찾아야 한다.

---

## 2. 자주 발생하는 오류 유형
- 반복문 범위 오류 (`<` vs `<=`)
- 변수 초기화 오류 (초기값 설정 문제)
- 잘못된 변수 사용
- 조건식 실수
- 경계값 처리 누락

---

## 3. 코딩테스트 디버깅 전략

### * 출력 기반 디버깅
```java
System.out.println(variable);
```
중간 값 변화를 출력하여 로직이 의도한 대로 실행되는지 확인한다.

### * 경계값 테스트
논리 오류는 대부분 경계값에서 발생한다.
- 최소 입력 (N=1)
- 최대 입력
- 음수 / 0 / 중복값
- 정렬 / 역정렬된 경우

---

## 4. 오답 발생 시 점검 순서
1. 경계값 테스트
2. 핵심 변수 출력
3. 조건식 및 초기값 확인
4. 시간 초과일 경우 → 시간복잡도 재검토

오류 유형에 따라 접근 방식을 달리해야 한다.
- `틀렸습니다` → 논리 오류 가능성
- `시간 초과` → 시간복잡도 문제 가능성

---

## 5. 핵심 정리
- 디버깅은 코드를 다시 짜는 것이 아니라 논리 흐름을 추적하는 과정이다.
- 오답 유형을 먼저 파악하면 디버깅 방향을 빠르게 설정할 수 있다.
- 문제를 많이 푸는 것만큼, 오류를 분석하는 경험도 중요하다.

---

## 6. 알게 된 점
- 오류 메시지나 오답 유형을 먼저 파악하면 디버깅 방향을 훨씬 빠르게 잡을 수 있다.  
- "틀렸습니다"와 "시간 초과"는 원인 자체가 다르기 때문에, 유형을 구분하는 것만으로도 불필요한 시간 낭비를 줄일 수 있다는 것을 배웠다.
