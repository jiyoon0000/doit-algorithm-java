# chapter01. 어떤 알고리즘으로 풀어야 할까?

# ⏱ Time Complexity(시간복잡도)

## 1. 정의

시간복잡도는 입력 크기 N이 증가할 때  
알고리즘의 연산 횟수가 어떻게 증가하는지를 나타내는 지표이다.

> 코딩테스트 예시 기준: 약 10⁸ 연산 ≒ 1초

---

## 2. Big-O 중심 분석

- **Big-O (O)**  
  상한(Upper Bound)을 나타내며,  
  일반적으로 최악의 경우(worst case)의 연산 횟수를 표현한다.  
  코딩테스트에서는 이 기준을 주로 사용한다.

- **Big-Omega (Ω)**  
  하한(Lower Bound)

- **Big-Theta (Θ)**  
  상한과 하한이 동일한 경우 (정확한 증가율)

---

## 3. 자주 등장하는 복잡도

| 복잡도 | 예시 |
|--------|------|
| O(1) | 배열 접근 |
| O(log N) | 이진 탐색 |
| O(N) | 선형 탐색 |
| O(N log N) | 병합 정렬 |
| O(N²) | 이중 반복문 |

---

## 4. 도출 원칙

- 상수 제거 → O(2N) → O(N)
- 최고차항만 남김 → O(N² + N) → O(N²)
- 반복문 구조 분석
- 분할 정복: 재귀 깊이 log N × 각 단계 N → O(N log N)

---

## 5. 입력 크기 기준 감각

| 입력 크기 | 허용 복잡도 |
|------------|--------------|
| ≤ 1,000 | O(N²) 가능 |
| ≤ 10,000 | O(N²) 가능 |
| ≤ 100,000 | O(N log N) |
| ≥ 1,000,000 | O(N) |

---

## 6. 알게된 점

- 시간복잡도는 코드를 작성한 뒤 계산하는 개념이 아니라  
  알고리즘 선택 단계에서 판단하는 기준으로 사용해야 한다.
- 입력 크기를 먼저 보고 가능한 복잡도를 추정하는 습관이 중요하다.
